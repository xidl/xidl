title: "插件开发"
description: "通过 JSON-RPC 插件协议为 xidlc 构建外部生成器。"
---

XIDL 支持外部生成器插件。插件本质上是一个由 `xidlc` 拉起并通过
JSON-RPC 2.0 通信的进程。

## 插件在编译链路中的位置

编译器仍然负责：

1. 解析
2. 诊断
3. HIR 构造
4. 文件落盘

插件只替换最后的“生成”阶段。

## 约定

- 可执行文件名：`xidl-<lang>`
- 调用形态：`xidlc gen --out-dir out <lang> path/to/file.idl`
- 传输：Unix 上优先本地 IPC，Windows 上优先回环 TCP
- 协议：通过 `--endpoint <uri>` 指定的 JSON-RPC 2.0

## 必需方法

### `parser_properties`

返回插件希望编译器启用的解析行为，例如：

```json
{ "expand_interface": true }
```

### `generate`

接收 HIR 和原始输入路径，返回一个生成文件列表。

## 实际开发流程

1. 选一个目标名，例如 `foo`
2. 构建名为 `xidl-foo` 的可执行文件
3. 实现 `parser_properties`
4. 实现 `generate`
5. 用样例 IDL 跑 `xidlc gen foo ...`
