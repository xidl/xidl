# Rust 集成

虽然 XIDL 本身不是 Rust 专属工具，但当前仓库里最成熟、最完整的落地方向
仍然集中在 Rust 生态。

## 两种常见接入方式

### 1. 手动运行 `xidlc`

这是推荐的新手路径。

优点：

- 过程直观
- 便于理解生成结果
- 出问题时更容易排查

例子：

```bash
xidlc -l rust -o src/generated api.idl
```

或者：

```bash
xidlc -l rust-axum -o src/generated api.idl
```

### 2. 用 `xidl-build` 接入 `build.rs`

当你已经稳定使用某个目标后，可以把生成流程接到 Cargo 里。

`Cargo.toml`：

```toml
[build-dependencies]
xidl-build = "<version>"
```

最小 `build.rs`：

```rust
fn main() {
    xidl_build::Builder::new()
        .with_lang("rust")
        .compile(&["api/example.idl"])
        .expect("generate xidl artifacts");
}
```

## 常见 Builder 选项

### 指定目标

```rust
let builder = xidl_build::Builder::new().with_lang("rust-axum");
```

### 指定输出目录

```rust
let builder = xidl_build::Builder::new().with_out_dir("src/generated");
```

### 只生成单文件 Schema 的文件名

```rust
let builder = xidl_build::Builder::new()
    .with_lang("openapi")
    .with_output_filename("city-api.json");
```

这一能力目前主要适合：

- `openapi`
- `openrpc`

### 控制 client / server

```rust
let builder = xidl_build::Builder::new()
    .with_lang("rust-axum")
    .with_client(true)
    .with_server(false);
```

## 推荐实践

- 学习阶段先直接用 `xidlc`
- 稳定后再决定是否引入 `xidl-build`
- 生成 HTTP 接口时，通常把 `rust-axum` 和 `openapi` 配对使用

## 和运行时的典型搭配

- `rust` -> 纯 Rust 数据类型
- `rust-axum` -> `xidl-rust-axum`
- `rust-jsonrpc` -> `xidl-jsonrpc`
