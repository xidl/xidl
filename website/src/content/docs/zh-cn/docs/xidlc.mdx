# 使用 `xidlc`

`xidlc` 是 XIDL 的标准入口。它读取一个或多个 `.idl` 文件，并为选定目标生成产物。

## 先建立正确心智模型

`xidlc` 的工作方式很简单：

1. 输入一份或多份 IDL
2. 选择一个目标生成器
3. 指定输出目录
4. 获得该目标对应的文件

最小命令：

```bash
xidlc gen --out-dir out rust path/to/file.idl
```

短写：

```bash
xidlc gen -o out rust path/to/file.idl
```

## 常用参数

- `--out-dir`, `-o`：指定输出目录
- `<lang>`：选择 `xidlc gen` 下的目标生成器子命令
- `--client`：为支持该参数的目标额外生成客户端相关产物
- `--server`：为支持该参数的目标生成服务端相关产物
- `--dry-run`：执行解析和生成流程，但不落盘

## 常见工作流

### 生成 Rust 类型

```bash
xidlc gen -o src/generated rust api.idl
```

### 生成 Axum HTTP 绑定

```bash
xidlc gen -o src/generated rust-axum api.idl
```

### 生成 JSON-RPC 绑定

```bash
xidlc gen -o src/generated rust-jsonrpc api.idl
```

### 生成 TypeScript REST 绑定

默认只生成客户端：

```bash
xidlc gen -o src/generated typescript-rest api.idl
```

只生成服务端：

```bash
xidlc gen -o src/generated typescript-rest --server api.idl
```

同时生成客户端和服务端：

```bash
xidlc gen -o src/generated typescript-rest --client --server api.idl
```

运行库和 Next.js 集成方式见 [TypeScript REST Server](../../rest/typescript-server/)。

### 生成 OpenAPI

```bash
xidlc gen -o generated openapi api.idl
```

### 生成 OpenRPC

```bash
xidlc gen -o generated openrpc api.idl
```

## 格式化

CLI 同时提供格式化子命令：

```bash
xidlc fmt path/to/file.idl
```

就地改写：

```bash
xidlc fmt --inplace path/to/file.idl
```
