title: "Plugin Development"
description: "Build an external generator for xidlc with the JSON-RPC plugin protocol."
---

XIDL supports external generators through a plugin model. A plugin is a process
that `xidlc` launches and talks to over JSON-RPC 2.0.

## How plugins fit

The compiler still owns:

1. parsing
2. diagnostics
3. HIR creation
4. output writing

Plugins only replace the final generation stage.

## Conventions

- executable name: `xidl-<lang>`
- invocation shape: `xidlc gen --out-dir out <lang> path/to/file.idl`
- transport: local IPC on Unix, loopback TCP on Windows
- protocol: JSON-RPC 2.0 with `--endpoint <uri>`

## Required methods

### `parser_properties`

Returns parser-side expectations, currently such as:

```json
{ "expand_interface": true }
```

### `generate`

Receives HIR plus the original input path and returns a list of generated
files.

## Practical workflow

1. choose a target name such as `foo`
2. build an executable named `xidl-foo`
3. implement `parser_properties`
4. implement `generate`
5. run `xidlc gen foo ...` against a sample IDL file
